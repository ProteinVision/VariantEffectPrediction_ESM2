# Variant Effect Prediction with ESM-2 (Zero-shot + Fine-tuning) Tutorial

Code and data for a hands-on tutorial on **using ESM-2 in Python for variant effect prediction (VEP)** on the GRB2 binding dataset.

🎥 Tutorial videos:
> - [Python walkthrough: zero-shot and fine-tuned VEP with ESM-2](https://www.youtube.com/@ProteinVision)

📚 References:
- ESM-2 paper: https://www.nature.com/articles/s41586-024-08203-9
- GRB2 DMS dataset source: https://github.com/gitter-lab/metl-pub/blob/main/data/dms_data/grb2-binding/grb2-binding.tsv

---

## 🔍 What this repository contains

- `VEP_with_ESM2.ipynb` – main Jupyter notebook used in the tutorial:
  - load and inspect the **GRB2 binding** mutation dataset (`variant`, `num_mutations`, `score`, `sequence`)
  - sample a balanced subset of **1,000 single** and **1,000 double** mutants
  - run **zero-shot mutation effect scoring** with masked-token probabilities from ESM-2
  - compute per-variant log-probability scores by summing `log(p(mut)/p(wt))` across mutation sites
  - compare experimental vs zero-shot scores for **all**, **single**, and **double** mutants (Pearson/Spearman)
  - split data into train/validation/test sets for supervised learning
  - fine-tune **ESM-2 + an MLP regression head** to predict continuous variant effect scores
  - evaluate model performance using **MSE**, **R²**, **Pearson r**, and **Spearman r**
- `environment.yml` – conda environment used for the notebook (Python, PyTorch, and `fair-esm`)
- `data/grb2_binding.tsv` – GRB2 mutation-effect dataset used in the notebook
- `sampled.csv` – generated by the notebook after sampling/scoring variants

## 🧪 Environment setup

1. **Clone the repository**

   ```bash
   git clone https://github.com/ProteinVision/VEP.git
   cd VEP
   ```

2. **Create and activate the conda environment**

   ```bash
   conda env create -f environment.yml
   conda activate ProteinVision_VEP
   ```

3. **Start JupyterLab**

   ```bash
   jupyter lab
   ```

4. **Open `VEP_with_ESM2.ipynb`** and run the cells from top to bottom.
